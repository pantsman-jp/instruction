## 1. これは何
Git のコマンドを解説します。

## 2. クローン から プルリクエスト まで
### 2.1. クローン する
ターミナルで、デスクトップまで移動します。

```
PS C:\Users\ユーザー名\desktop>
```

このようになっていれば良いです。

そして、クローンしたいリポジトリの URL をコピーし、以下をターミナルで実行します。

```
git clone リポジトリのURL
```

そして、以下のコマンドを順に行い、作業フォルダを VSCode で開きます。

```
cd リポジトリ名
code .
```

作業しましょう。作業したら、`CHANGELOG.md` にその内容を書くことを忘れずに。

### バージョン管理
`CHANGELOG.md` にはバージョンを表記します。
[PBL-Game](https://github.com/pantsman-jp/PBL-Game/tree/main) では以下のフォーマットで揃えています。

```Markdown
## vA.B.C (YYYY-MM-DD)
- 変更内容...
```

バージョン管理は [セマンティックバージョニング](https://semver.org/lang/ja/) に準拠しています。

ざっくりいうと、

- `A` : 大幅な、不可逆的な変更を加えたときに +1
- `B` : 機能追加をしたときに +1
- `C` : バグ修正したときに +1

です。参考：<https://qiita.com/Jirox/items/5d74dbaafc86dfb02cdb>

バージョンは小数点数ではないので、繰り上がりはないです。
たとえば、`v1.9.2` に機能追加をしたとすると、バージョンは `v1.10.0` となります。

## 2.2. ブランチを作る
ファイルの変更が済んだら、ブランチを作ってから、コミットをします。

新しいブランチを作るには、ターミナルで以下を実行します。

```
git checkout -b 新しいブランチ名
```

ブランチ名は作業内容が分かるようにつけましょう。

一応、今自分がどのブランチで作業しているのか確認してみましょう。

```
git branch
```

さっき作ったブランチにいれば、OK です。

## 2.3. コミット する
ファイルの変更が済んだら、コミットをしましょう。

コミットの前には変更したファイルをステージング（変更の記録を残すファイルを選ぶ）する必要があります。
以下を実行しましょう。

```
git add .
```

これはすべてのファイルを記録に残す、という意味です。
もちろん、特定のファイルのみをステージングすることもできます。
以下のように。

```
git add README.md
```

では、コミットしてみましょう。

```
git commit -m "コミットメッセージ"
```

これでリポジトリに最初の変更履歴が記録されます。

コミットメッセージは簡潔でわかりやすいものが良いです。一言で変更内容がわかるようにしましょう。

## 2.4 プッシュ と プルリクエスト
コミットが済んだらプッシュしましょう。以下をターミナルで実行するだけです。

```
git push --set-upstream origin さっき作ったブランチ名
```

これで、プッシュは完了です。

次にプルリクエストをしましょう。GitHub のリポジトリページに行きましょう。
すると、`Compare & pull request` という緑のボタンがあるはずです。それを押しましょう。

タイトルをつけ、変更内容を書きます。書けたら、右下にある `Create pull request` という緑のボタンを押します。

これで、ファイルの変更をプルリクエストすることができました。
リポジトリの管理者がこれを見て、マージするか、改善点を言ってくることでしょう。
