## 1. これは何

共同開発で必須となる Git の基本的なコマンドと、GitHub 上でのプルリクエスト（PR）作成までの流れを解説します。

## 2. 基本ワークフロー：クローンからプルリクエストまで

### 2.1. リポジトリをクローンする

まずはリモート（GitHub 上）のリポジトリを自分の PC にコピーします。
ターミナルでデスクトップなど作業したい場所に移動し、以下を実行してください。

```PowerShell
git clone git@github.com:ユーザー名/リポジトリ名.git
```

クローンが完了したら、作成されたディレクトリに移動し、VSCode で開きます。

```PowerShell
cd リポジトリ名
code .
```

### 2.2. 作業用ブランチを作成する

**重要：コードを書き始める前に必ず新しいブランチを作成してください。** `main` ブランチで直接作業すると、他の人の変更と衝突しやすくなります。

作業内容が伝わる名前（例：`add-login-feature`）をつけて作成・移動します。

```PowerShell
git checkout -b ブランチ名
```

現在どのブランチにいるかは以下のコマンドで確認できます。

```PowerShell
git branch
```

### 2.3. 変更を記録（コミット）する

ファイルを作成・編集したら、その変更を履歴として保存します。
まず、変更したファイルを「ステージング領域」に追加します。

```PowerShell
# すべての変更を追加する場合
git add .

# 特定のファイル（例: CHANGELOG.md）のみ追加する場合
git add CHANGELOG.md
```

次に、ステージングされた変更を確定（コミット）させます。

```PowerShell
git commit -m "コミットメッセージ（例：更新履歴の追加）"
```
※ `CHANGELOG.md` 等への記述が必要な場合は、コミット前に忘れず記入しましょう。

### 2.4. プッシュとプルリクエスト

手元の変更履歴を GitHub に送信（プッシュ）します。

```PowerShell
git push -u origin ブランチ名
```
※ `-u` オプションをつけることで、次回以降は `git push` だけで送信できるようになります。

プッシュ完了後、GitHub のリポジトリページを開くと `Compare & pull request` というボタンが表示されます。

1. ボタンを押し、タイトルと具体的な変更内容を記述します。
2. `Create pull request` をクリックして完了です。

リポジトリの管理者が内容を確認し、問題がなければ `Merge`（取り込み）が行われます。
